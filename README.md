# Планировщик путешествий

Консольное приложение на C# для поиска и оптимизации маршрутов между городами.
Проект включает полный цикл разработки: проектирование, реализацию алгоритмов, модульное и структурное тестирование.

## Технологический стек
*   Язык: C# (.NET)
*   Тестирование: MSTest Framework
*   Методологии: ООП, структурное тестирование

## Функциональность
*   Ввод параметров: города, даты, предпочтения (время, пересадки, бюджет).
*   Построение маршрутов: реализован алгоритм поиска в глубину (DFS) с ограничением по количеству пересадок.
*   Оптимизация: выбор оптимального маршрута по времени или стоимости с помощью линейного поиска.
*   Визуализация: вывод детальной информации о маршруте в консоль.

## Архитектура
Приложение построено на объектно-ориентированной архитектуре:
*   Класс `Path` — элементарный сегмент пути.
*   Класс `Route` — полный маршрут, состоящий из нескольких сегментов.
*   Класс `Planner` — ядро системы, содержащее алгоритмы поиска и оптимизации.
*   Класс `Preferences` — контейнер для пользовательских ограничений.
*   Класс `Program` — точка входа и консольный интерфейс.

## Тестирование и качество кода
Проведено комплексное тестирование ключевого метода `FindBestRoute`:
*   **Структурное тестирование:** Построен потоковый граф метода, рассчитана цикломатическая сложность (V(G) = 6).
*   **Модульное тестирование:** Разработано 7 тестовых случаев, покрывающих основные и граничные сценарии (пустой список, null, сравнение по разным критериям).
*   **Отладка:** Выявлена и исправлена ошибка — отсутствие проверки входного параметра на `null`, что приводило к исключению `NullReferenceException`.
